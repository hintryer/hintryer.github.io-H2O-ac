<h2 id="详细步骤">详细步骤</h2>

<h3 id="初始化">初始化</h3>

<h4 id="定位工作目录">定位工作目录</h4>

<p>打开 Git Bash 终端，使用 <code class="language-plaintext highlighter-rouge">cd</code>命令加上要切换的路径，例如：<code class="language-plaintext highlighter-rouge">cd /c/Users/username/Documents</code>；按下回车键，即可切换到指定的路径。需要注意的是路径中需要使用正斜杠 / 分隔路径，而不是反斜杠 \。</p>

<p>使用<code class="language-plaintext highlighter-rouge">pwd</code>可以显示当前路径</p>

<h4 id="初始化用户信息">初始化用户信息</h4>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git config --global user.name 'better'
git config --global user.email '1063266220@qq.com'
</code></pre></div></div>

<h3 id="创建本地仓库">创建本地仓库</h3>

<p>创建本地仓库的方法有两种：</p>

<ul>
  <li>一种是创建全新的仓库：<code class="language-plaintext highlighter-rouge">git init</code>，会在当前目录初始化创建仓库，也可指定目录：<code class="language-plaintext highlighter-rouge">git init [文件目录]</code>执行命令后，如下显示</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/
</code></pre></div></div>

<p>创建完多出了一个被隐藏的.git 目录，这就是本地仓库 Git 的工作场所，而且是一个空的仓库（empty Git repository）。没事千万不要手动修改这个目录里面的文件，不然改乱了，就把 Git 仓库给破坏了。</p>

<ul>
  <li>另一种是克隆远程仓库：<code class="language-plaintext highlighter-rouge">git clone [url]</code>
克隆远程仓库，如在 github 上创建的仓库“<code class="language-plaintext highlighter-rouge">https://github.com/kwonganding/KWebNote.git</code>”</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git clone 'https://github.com/kwonganding/KWebNote.git'
Cloning into 'KWebNote'...
remote: Enumerating objects: 108, done.
remote: Counting objects: 100% (108/108), done.
remote: Compressing objects: 100% (60/60), done.
remote: Total 108 (delta 48), reused 88 (delta 34), pack-reused 0
Receiving objects: 100% (108/108), 9.36 KiB | 736.00 KiB/s, done.
Resolving deltas: 100% (48/48), done.
</code></pre></div></div>

<p>会在当前目录下创建“KWebNote”项目目录。</p>

<h3 id="暂存区-add">暂存区 add</h3>

<p>把工作区变化放到暂存区中，执行命令<code class="language-plaintext highlighter-rouge">git add .</code>可以一次性把所有变化文件放入暂存区</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git add .
</code></pre></div></div>

<p>执行上面的命令，没有任何显示，说明添加成功。也可以先执行<code class="language-plaintext highlighter-rouge">git rm --cached &lt;file&gt;</code>命令将指定的<code class="language-plaintext highlighter-rouge">file</code>从当前的暂存区中删除但保留文件在本地工作目录中。
| <strong>指令</strong> | <strong>描述</strong> |
| — | — |
| git add [file1] [file2] | 添加文件到暂存区，包括修改的文件、新增的文件 |
| git add [dir] | 同上，添加目录到暂存区，包括子目录 |
| git add . | 同上，添加<strong>所有</strong>修改、新增文件（未跟踪）到暂存区 |
| git rm [file] | 删除工作区文件，并且将这次删除放入暂存区 |
|git rm –cached [file]|删除放入暂存区 ，但该文件会保留在工作区|</p>

<h3 id="提交-commit-记录">提交 commit-记录</h3>

<p>把暂存区内容提交到版本库，命令如下（此处文字说明可以不加引号）</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git commit -m '提交的内容说明'
</code></pre></div></div>

<p>执行上面的命令，如下显示</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git commit -m "wrote a readme file"
[master (root-commit) eaadf4e] wrote a readme file
 1 file changed, 2 insertions(+)
 create mode 100644 readme.txt
</code></pre></div></div>

<p><code class="language-plaintext highlighter-rouge">git commit</code>命令执行成功后会告诉你，<code class="language-plaintext highlighter-rouge">1 file changed</code>：1 个文件被改动（我们新添加的 readme.txt 文件）；<code class="language-plaintext highlighter-rouge">2 insertions</code>：插入了两行内容（readme.txt 有两行内容）。
以上命令相当于存档了一次，在版本库中产生一次提交记录并生成版本号。本次存档，不耽误我们在工作区 (项目文件夹) 下继续编写项目</p>

<p>如果改的代码过多，忘记改过哪些了，可以运行<code class="language-plaintext highlighter-rouge">git status</code>命令来查看 git 仓库变化，只能看未提交的所有变更的文件状态
| <strong>指令</strong> | <strong>描述</strong> |
| — | — |
| git commit -m ‘说明’ | 提交变更，参数<code class="language-plaintext highlighter-rouge">-m</code>设置提交的描述信息，应该正确提交，不带该参数会进入说明编辑模式 |
| git commit -a | 参数<code class="language-plaintext highlighter-rouge">-a</code>，表示直接从工作区提交到版本库，略过了<code class="language-plaintext highlighter-rouge">git add</code>步骤，不包括新增的文件 |
| git commit [file] | 提交暂存区的指定文件到仓库区 |
| git commit –amend -m | 使用一次新的<code class="language-plaintext highlighter-rouge">commit</code>，替代上一次提交，会修改<code class="language-plaintext highlighter-rouge">commit</code>的<code class="language-plaintext highlighter-rouge">hash</code>值（id） |
| git <strong>reflog</strong> | 查看所有可用的历史版本记录（实际是 HEAD 变更记录），包含被回退的记录（<strong>重要</strong>） |
| git status | 查看本地仓库状态，比较常用的指令，加参数<code class="language-plaintext highlighter-rouge">-s</code>简洁模式 |</p>

<h3 id="将本地代码仓库关联到-gitee-上git-remote-add">将本地代码仓库关联到 gitee 上<code class="language-plaintext highlighter-rouge">git remote add</code>
</h3>

<ul>
  <li>第一次提交直接执行以下命令关联</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code> git remote add origin 仓库地址
</code></pre></div></div>

<p>此后，每次本地提交后，只要有必要，就可以使用命令<code class="language-plaintext highlighter-rouge">git push origin master</code>推送最新修改；</p>

<ul>
  <li>第二次提交先删除，再关联</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote rm origin
git remote add origin 仓库地址
</code></pre></div></div>

<p>否则会在使用命令<code class="language-plaintext highlighter-rouge">git remote add</code>时报错：<code class="language-plaintext highlighter-rouge">fatal: remote origin already exists.</code>，说明本地库已经关联了一个名叫<code class="language-plaintext highlighter-rouge">origin</code>的远程库，</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git pull --rebase origin master

<h2 id="git-add-报错">git add .报错</h2>

<p>git add .报错 <code class="language-plaintext highlighter-rouge">warning: adding embedded git repository\:xxxxxxxxxx</code></p>

<p>执行- $ git add .命令时出现出现这个报错</p>

<div class="language-bash highlighter-rouge"><div class="highlight"><pre class="highlight"><code>warning: adding embedded git repository:xxxxxxxxxx
hint: You<span class="s1">'ve added another git repository inside your current repository.
hint: Clones of the outer repository will not contain the contents of
hint: the embedded repository and will not know how to obtain it.
hint: If you meant to add a submodule, use:
hint:
hint: git submodule add &lt;url&gt;xxxxxxxxxx
hint:
hint: If you added this path by mistake, you can remove it from the
hint: index with:
hint:
hint: git rm --cached   xxxx
hint:
hint: See "git help submodule" for more information.
</span></code></pre></div></div>

<p><strong>原因：</strong> 使用 <code class="language-plaintext highlighter-rouge">git add .</code> 时，出现上述错误。是因为在当前 <code class="language-plaintext highlighter-rouge">git</code> 仓库中同时包含有另一个<code class="language-plaintext highlighter-rouge">git</code>仓库。如当前仓库目录下的子文件夹内又是一个仓库。</p>

<p><strong>解决：</strong> 删除子文件夹的.git 文件， 重新执行<code class="language-plaintext highlighter-rouge">git add .</code></p>

<h2 id="git-remote-add时报错">
<code class="language-plaintext highlighter-rouge">git remote add</code>时报错：</h2>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote add origin git@gitee.com:liaoxuefeng/learngit.git
fatal: remote origin already exists.
</code></pre></div></div>

<p>这说明本地库已经关联了一个名叫<code class="language-plaintext highlighter-rouge">origin</code>的远程库，可以用<code class="language-plaintext highlighter-rouge">git remote rm &lt;name&gt;</code>命令已有的远程库。</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote -v
origin	git@github.com:michaelliao/learngit.git (fetch)
origin	git@github.com:michaelliao/learngit.git (push)
</code></pre></div></div>

<p>可以看到，本地库已经关联了<code class="language-plaintext highlighter-rouge">origin</code>的远程库，并且，该远程库指向 GitHub。我们可以删除已有的 GitHub 远程库再关联 Gitee 的远程库</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote rm origin
git remote add origin 仓库地址
</code></pre></div></div>

<h2 id="git-上传报错">git 上传报错</h2>

<h3 id="第一个问题">第一个问题</h3>

<p>上传步骤：<br>
<code class="language-plaintext highlighter-rouge">git add .</code><br>
<code class="language-plaintext highlighter-rouge">git commit -m "提示消息"</code><br>
<code class="language-plaintext highlighter-rouge">git push origin master</code></p>

<p>出错：<br>
<code class="language-plaintext highlighter-rouge">! [rejected] master -&gt; master (fetch first) error: failed to push some refs to ' 。。。'</code></p>

<p>出现这个问题是因为 github 中的 README.md 文件不在本地代码目录中，可以通过如下命令进行代码合并</p>

<p><code class="language-plaintext highlighter-rouge">git pull --rebase origin master</code></p>

<h3 id="第二个问题">第二个问题</h3>

<p><code class="language-plaintext highlighter-rouge">! [remote rejected] master -&gt; master (pre-receive hook declined)</code></p>

<p>在推送代码时报错如上，网上搜到的方法说是项目的 setting 中 master 是受保护的项目，所以开发者无法推送，解决方法是修改 protected 的设置，或者新建一个分支，推送到自己的分支上</p>

<p><a href="https://www.cnblogs.com/cppeterpan/p/7289266.html" rel="nofollow" target="_blank" class="extlinks">解决办法的文章</a></p>

<h2 id="git-push-报错">git push 报错</h2>

<p>gitee 第一次提交代码提交不上去，到 push 这一步老是遇到</p>

<div class="language-javascript highlighter-rouge"><div class="highlight"><pre class="highlight"><code>  <span class="o">!</span> <span class="err">\</span><span class="p">[</span><span class="nx">remote</span> <span class="nx">rejected</span><span class="err">\</span><span class="p">]</span> <span class="nx">master</span> <span class="o">-&gt;</span> <span class="nf">master </span><span class="p">(</span><span class="nx">hook</span> <span class="nx">declined</span><span class="p">)</span>

<p>这种情况，老是提不上去。那是因为。你的邮箱设置，<strong>不公开我的邮箱</strong>把这个勾上了，取消勾选就行了。这样导致找不到你的远程地址。所以推不上去</p>

                <h2 id="总结">总结</h2>

<h2 id="详细步骤">详细步骤</h2>

<h3 id="初始化">初始化</h3>

<h4 id="定位工作目录">定位工作目录</h4>

<p>打开 Git Bash 终端，使用 <code class="language-plaintext highlighter-rouge">cd</code>命令加上要切换的路径，例如：<code class="language-plaintext highlighter-rouge">cd /c/Users/username/Documents</code>；按下回车键，即可切换到指定的路径。需要注意的是路径中需要使用正斜杠 / 分隔路径，而不是反斜杠 \。</p>

<p>使用<code class="language-plaintext highlighter-rouge">pwd</code>可以显示当前路径</p>

<h4 id="初始化用户信息">初始化用户信息</h4>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git config --global user.name 'better'
git config --global user.email '1063266220@qq.com'
</code></pre></div></div>

<h3 id="创建本地仓库">创建本地仓库</h3>

<p>创建本地仓库的方法有两种：</p>

<ul>
  <li>一种是创建全新的仓库：<code class="language-plaintext highlighter-rouge">git init</code>，会在当前目录初始化创建仓库，也可指定目录：<code class="language-plaintext highlighter-rouge">git init [文件目录]</code>执行命令后，如下显示</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/
</code></pre></div></div>

<p>创建完多出了一个被隐藏的.git 目录，这就是本地仓库 Git 的工作场所，而且是一个空的仓库（empty Git repository）。没事千万不要手动修改这个目录里面的文件，不然改乱了，就把 Git 仓库给破坏了。</p>

<ul>
  <li>另一种是克隆远程仓库：<code class="language-plaintext highlighter-rouge">git clone [url]</code>
克隆远程仓库，如在 github 上创建的仓库“<code class="language-plaintext highlighter-rouge">https://github.com/kwonganding/KWebNote.git</code>”</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git clone 'https://github.com/kwonganding/KWebNote.git'
Cloning into 'KWebNote'...
remote: Enumerating objects: 108, done.
remote: Counting objects: 100% (108/108), done.
remote: Compressing objects: 100% (60/60), done.
remote: Total 108 (delta 48), reused 88 (delta 34), pack-reused 0
Receiving objects: 100% (108/108), 9.36 KiB | 736.00 KiB/s, done.
Resolving deltas: 100% (48/48), done.
</code></pre></div></div>

<p>会在当前目录下创建“KWebNote”项目目录。</p>

<h3 id="暂存区-add">暂存区 add</h3>

<p>把工作区变化放到暂存区中，执行命令<code class="language-plaintext highlighter-rouge">git add .</code>可以一次性把所有变化文件放入暂存区</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git add .
</code></pre></div></div>

<p>执行上面的命令，没有任何显示，说明添加成功。也可以先执行<code class="language-plaintext highlighter-rouge">git rm --cached &lt;file&gt;</code>命令将指定的<code class="language-plaintext highlighter-rouge">file</code>从当前的暂存区中删除但保留文件在本地工作目录中。
| <strong>指令</strong> | <strong>描述</strong> |
| — | — |
| git add [file1] [file2] | 添加文件到暂存区，包括修改的文件、新增的文件 |
| git add [dir] | 同上，添加目录到暂存区，包括子目录 |
| git add . | 同上，添加<strong>所有</strong>修改、新增文件（未跟踪）到暂存区 |
| git rm [file] | 删除工作区文件，并且将这次删除放入暂存区 |
|git rm –cached [file]|删除放入暂存区 ，但该文件会保留在工作区|</p>

<h3 id="提交-commit-记录">提交 commit-记录</h3>

<p>把暂存区内容提交到版本库，命令如下（此处文字说明可以不加引号）</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git commit -m '提交的内容说明'
</code></pre></div></div>

<p>执行上面的命令，如下显示</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>$ git commit -m "wrote a readme file"
[master (root-commit) eaadf4e] wrote a readme file
 1 file changed, 2 insertions(+)
 create mode 100644 readme.txt
</code></pre></div></div>

<p><code class="language-plaintext highlighter-rouge">git commit</code>命令执行成功后会告诉你，<code class="language-plaintext highlighter-rouge">1 file changed</code>：1 个文件被改动（我们新添加的 readme.txt 文件）；<code class="language-plaintext highlighter-rouge">2 insertions</code>：插入了两行内容（readme.txt 有两行内容）。
以上命令相当于存档了一次，在版本库中产生一次提交记录并生成版本号。本次存档，不耽误我们在工作区 (项目文件夹) 下继续编写项目</p>

<p>如果改的代码过多，忘记改过哪些了，可以运行<code class="language-plaintext highlighter-rouge">git status</code>命令来查看 git 仓库变化，只能看未提交的所有变更的文件状态
| <strong>指令</strong> | <strong>描述</strong> |
| — | — |
| git commit -m ‘说明’ | 提交变更，参数<code class="language-plaintext highlighter-rouge">-m</code>设置提交的描述信息，应该正确提交，不带该参数会进入说明编辑模式 |
| git commit -a | 参数<code class="language-plaintext highlighter-rouge">-a</code>，表示直接从工作区提交到版本库，略过了<code class="language-plaintext highlighter-rouge">git add</code>步骤，不包括新增的文件 |
| git commit [file] | 提交暂存区的指定文件到仓库区 |
| git commit –amend -m | 使用一次新的<code class="language-plaintext highlighter-rouge">commit</code>，替代上一次提交，会修改<code class="language-plaintext highlighter-rouge">commit</code>的<code class="language-plaintext highlighter-rouge">hash</code>值（id） |
| git <strong>reflog</strong> | 查看所有可用的历史版本记录（实际是 HEAD 变更记录），包含被回退的记录（<strong>重要</strong>） |
| git status | 查看本地仓库状态，比较常用的指令，加参数<code class="language-plaintext highlighter-rouge">-s</code>简洁模式 |</p>

<h3 id="将本地代码仓库关联到-gitee-上git-remote-add">将本地代码仓库关联到 gitee 上<code class="language-plaintext highlighter-rouge">git remote add</code>
</h3>

<ul>
  <li>第一次提交直接执行以下命令关联</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code> git remote add origin 仓库地址
</code></pre></div></div>

<p>此后，每次本地提交后，只要有必要，就可以使用命令<code class="language-plaintext highlighter-rouge">git push origin master</code>推送最新修改；</p>

<ul>
  <li>第二次提交先删除，再关联</li>
</ul>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote rm origin
git remote add origin 仓库地址
</code></pre></div></div>

<p>否则会在使用命令<code class="language-plaintext highlighter-rouge">git remote add</code>时报错：<code class="language-plaintext highlighter-rouge">fatal: remote origin already exists.</code>，说明本地库已经关联了一个名叫<code class="language-plaintext highlighter-rouge">origin</code>的远程库，</p>

<h3 id="将代码由本地仓库上传到-gitee-远程仓库">将代码由本地仓库上传到 gitee 远程仓库</h3>

<p>执行以下命令，先拉取后推送</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git pull --rebase origin master
git push -u origin master
</code></pre></div></div>

<h2 id="git-add-报错">git add .报错</h2>

<p>git add .报错 <code class="language-plaintext highlighter-rouge">warning: adding embedded git repository\:xxxxxxxxxx</code></p>

<p>执行- $ git add .命令时出现出现这个报错</p>

<div class="language-bash highlighter-rouge"><div class="highlight"><pre class="highlight"><code>warning: adding embedded git repository:xxxxxxxxxx
hint: You<span class="s1">'ve added another git repository inside your current repository.
hint: Clones of the outer repository will not contain the contents of
hint: the embedded repository and will not know how to obtain it.
hint: If you meant to add a submodule, use:
hint:
hint: git submodule add &lt;url&gt;xxxxxxxxxx
hint:
hint: If you added this path by mistake, you can remove it from the
hint: index with:
hint:
hint: git rm --cached   xxxx
hint:
hint: See "git help submodule" for more information.
</span></code></pre></div></div>

<p><strong>原因：</strong> 使用 <code class="language-plaintext highlighter-rouge">git add .</code> 时，出现上述错误。是因为在当前 <code class="language-plaintext highlighter-rouge">git</code> 仓库中同时包含有另一个<code class="language-plaintext highlighter-rouge">git</code>仓库。如当前仓库目录下的子文件夹内又是一个仓库。</p>

<p><strong>解决：</strong> 删除子文件夹的.git 文件， 重新执行<code class="language-plaintext highlighter-rouge">git add .</code></p>

<h2 id="git-remote-add时报错">
<code class="language-plaintext highlighter-rouge">git remote add</code>时报错：</h2>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote add origin git@gitee.com:liaoxuefeng/learngit.git
fatal: remote origin already exists.
</code></pre></div></div>

<p>这说明本地库已经关联了一个名叫<code class="language-plaintext highlighter-rouge">origin</code>的远程库，可以用<code class="language-plaintext highlighter-rouge">git remote rm &lt;name&gt;</code>命令已有的远程库。</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote -v
origin	git@github.com:michaelliao/learngit.git (fetch)
origin	git@github.com:michaelliao/learngit.git (push)
</code></pre></div></div>

<p>可以看到，本地库已经关联了<code class="language-plaintext highlighter-rouge">origin</code>的远程库，并且，该远程库指向 GitHub。我们可以删除已有的 GitHub 远程库再关联 Gitee 的远程库</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>git remote rm origin
git remote add origin 仓库地址
</code></pre></div></div>

<h2 id="git-上传报错">git 上传报错</h2>

<h3 id="第一个问题">第一个问题</h3>

<p>上传步骤：<br>
<code class="language-plaintext highlighter-rouge">git add .</code><br>
<code class="language-plaintext highlighter-rouge">git commit -m "提示消息"</code><br>
<code class="language-plaintext highlighter-rouge">git push origin master</code></p>

<p>出错：<br>
<code class="language-plaintext highlighter-rouge">! [rejected] master -&gt; master (fetch first) error: failed to push some refs to ' 。。。'</code></p>

<p>出现这个问题是因为 github 中的 README.md 文件不在本地代码目录中，可以通过如下命令进行代码合并</p>

<p><code class="language-plaintext highlighter-rouge">git pull --rebase origin master</code></p>

<h3 id="第二个问题">第二个问题</h3>

<p><code class="language-plaintext highlighter-rouge">! [remote rejected] master -&gt; master (pre-receive hook declined)</code></p>

<p>在推送代码时报错如上，网上搜到的方法说是项目的 setting 中 master 是受保护的项目，所以开发者无法推送，解决方法是修改 protected 的设置，或者新建一个分支，推送到自己的分支上</p>

<p><a href="https://www.cnblogs.com/cppeterpan/p/7289266.html" rel="nofollow" target="_blank" class="extlinks">解决办法的文章</a></p>

<h2 id="git-push-报错">git push 报错</h2>

<p>gitee 第一次提交代码提交不上去，到 push 这一步老是遇到</p>

<div class="language-javascript highlighter-rouge"><div class="highlight"><pre class="highlight"><code>  <span class="o">!</span> <span class="err">\</span><span class="p">[</span><span class="nx">remote</span> <span class="nx">rejected</span><span class="err">\</span><span class="p">]</span> <span class="nx">master</span> <span class="o">-&gt;</span> <span class="nf">master </span><span class="p">(</span><span class="nx">hook</span> <span class="nx">declined</span><span class="p">)</span>
<span class="nx">error</span><span class="p">:</span> <span class="nx">failed</span> <span class="nx">to</span> <span class="nx">push</span> <span class="nx">some</span> <span class="nx">refs</span> <span class="nx">to</span>
</code></pre></div></div>

<p>这种情况，老是提不上去。那是因为。你的邮箱设置，<strong>不公开我的邮箱</strong>把这个勾上了，取消勾选就行了。这样导致找不到你的远程地址。所以推不上去</p>
